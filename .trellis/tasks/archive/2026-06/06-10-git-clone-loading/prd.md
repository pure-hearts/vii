# Add loading spinner during git clone

## 目标 (Goal)

在执行 `vii init` 交互式克隆项目或自定义拉取模板时，由于 `git clone` 比较耗时且主进程阻塞导致终端无错觉响应，需要在克隆期间在终端显示一个高反馈、健壮的 Loading 状态。本任务将引入 `ora`，深度重构克隆底层的异步管道流逻辑，并在此基础上加入进度/网速解析、超时保护、自动回退、进程信号清理以及 CI 日志降级，提供顶级的终端交互体验。

## 需求 (Requirements)

1. **终端 Loading 提示 (使用 Ora)**：使用 `ora` 库实现色彩绚丽的动态 Spinner，智能提取出仓库的 `owner/repo` 与分支标签，动态输出类似于 `正在下载 owner/repo (分支: vue-pc)...` 的提示信息。
2. **异步执行 Git 克隆**：将原有的同步 `child_process.execSync` 重构为 `child_process.spawn` 并包装为 `Promise`，以非阻塞方式运行 `git clone`。同时**必须**显式传入 `--progress` 参数，确保非 TTY/重定向管道模式下 Git 仍输出进度数据。
3. **实时进度百分比、网速与数据量提取**：
   - 监听 Git 子进程的 `stderr` 输出流。
   - 用正则表达式匹配 Git 输出的实时进度（百分比、已下载大小、下载速率）。
   - 将其解析并渲染到 Spinner 后方（如 `... 45% (1.56 MiB, 512 KiB/s) (通过 KKGitHub 加速)`）。
4. **终端更新节流控制**：限制 Spinner 终端刷新率（如最小更新周期 100ms），避免 Git 高频输出导致某些控制台闪烁或卡顿。
5. **CI 模式与非 TTY 降级**：
   - 自动检测当前执行环境是否为 CI（`process.env.CI`）或非交互式终端（`!process.stdout.isTTY`）。
   - 在此环境下停用动态 Spinner，降级为单行静态日志打印（防止生成包含大量 `\r` 字符的日志文件）。
6. **耗时统计与彩色图标反馈**：
   - **成功**：克隆成功结束后，自动计算耗时，展示绿色勾号，输出如 `✔ 下载 owner/repo 成功 (耗时 3.5s)`。
   - **失败**：克隆彻底失败后，展示红色叉号，输出如 `✖ 下载 owner/repo 失败`。
7. **Git 克隆超时保护 (Timeout)**：为克隆子进程设置 45 秒超时机制。超时若仍未完成，强制 `kill` 子进程并报错，提示用户更换镜像或检查网络。
8. **镜像克隆失败自动降级重试 (Auto-Fallback)**：
   - 若用户通过加速镜像源克隆失败，程序不直接崩溃。
   - 自动在 Spinner 中给出警告提示，然后抹除镜像后缀，重新发起一次针对 GitHub 官方源的异步克隆。
   - 若官方源重试依然失败，或者原本就没有使用镜像源直接失败，才最终判定为失败并抛出异常。
9. **智能诊断引导与进程优雅清理**：
   - 捕获 `spawn` 期间 Git 输出的所有 `stderr` 诊断报错，并在报错后给出诊断引导（如运行 `vii speed` 或检查镜像配置）。
   - 注册主进程 `SIGINT` (Ctrl+C) 监听器，确保当用户中途终止时，自动杀死 Git 子进程并彻底清理产生的临时目录。
   - 注意：在正常结束或报错退出时，必须完全移除 `SIGINT` 等事件监听器以防止内存泄露。

## 验收标准 (Acceptance Criteria)

- [ ] **交互式进度**：交互式克隆期间，终端显示实时百分比、下载量及网速进度，刷新流畅不闪烁。
- [ ] **元数据展示**：终端显示当前正在克隆的 `owner/repo` 以及分支名称、所用镜像名称。
- [ ] **Metrics 统计**：克隆结束后，显示最终耗时和成功状态（如 `✔ 下载 vfiee/project-boilerplate 成功 (耗时 4.2s)`）。
- [ ] **重试自动回退**：使用故障/无效镜像源（或制造网络超时），能够成功触发“降级回退到官方源重试”的逻辑并给出 `⚠️` 警告。
- [ ] **环境降级**：在非 TTY（如 `pnpm test` 或 CI 执行）时，自动屏蔽 Spinner，降级为干净的静态文本输出。
- [ ] **诊断输出**：连续失败或超时退出时，终端光标恢复，显示红色的错误 icon、底层 Git 的 stderr 错误流以及自助排查提示。
- [ ] **强退清理**：中途按 `Ctrl+C` 强退，临时文件夹被干净清除且没有后台僵尸 `git clone` 进程。
- [ ] **测试通过**：全套单元测试用例 `pnpm --filter @vyron/cli test` 依然 100% 成功通过。
